合同-订单管理后台 (Axure)

为了增加销售和教学流程的灵活性，将ERP原先的订单系统升级为合同-订单系统。负责产品设计、研发协调和验收上线，研发周期近两个月，可能是我做过的第一个真正意义上的大需求。

礼品二期 (Axure)

包含用于收集邮寄地址的前台页面，和为运营&供应商提供的礼品管理后台。之所以叫做礼品二期，是因为最终为了节约研发成本，只做为一期需求上线了ERP部分。

APP 语音检测 (Axure)

在当前公司做的唯一一个纯移动端需求。流程和页面都比较简单，文档中带交互原型。可惜最终因为移动端版本计划变更需求被冻结。

邀请管理+兑现管理后台 (Axure)

配合运营部门一项长期活动做的两个后台页面。其中邀请管理的业务逻辑非常复杂，已上线运行，暂未出现问题。兑现管理尚在测试中。

合作方查询后台 (Axure)

提供给合作方查询渠道转化数据的简单后台。目前正处于测试阶段。

攻略管理后台 (Axure)

刚入职时接手的工作，根据已有的度假攻略 web 前台原型做出配套的管理后台原型。

状态：已上线

APP 晒单 (Sketch, HTML/CSS)

原名「动态」,包含一套 APP 页面 + WEB 承接页 + 管理后台。试图通过增加社交属性来提高 app 的活跃度。功能开发初期，由于缺少前端，web 承接页的静态页面是由我来代为现实。

V1.2 将新增短视频功能，原型制作过程中尝试学习使用 Framer 实现交互动画。原型文档尚未完成（页面跳转逻辑未标注）。

状态：已上线

春节&元宵活动 (Sketch)

负责春节助力活动和元宵猜价活动的内容策划、文案、页面原型，活动完成后的数据分析和总结。

状态：已完成

WEB 首页改版 (Sketch)

对首页布局重新排版，优化导航栏/推广栏交互逻辑，并重新设计了别墅产品卡片cell的内容，以期降低首页跳出率。同时为将来的 PCweb 和 mweb 合并，向响应式设计演进过渡做准备。

状态：部分已上线

积分管理后台 (Axure)

满足客服和营销部门的需求，用于活动奖券的人工发放。这是一个逻辑非常简单的原型，设计中多照顾到程序实现的便捷性。

状态：已上线

转账汇款页 (Axure)

由于业务性质，平台的订单均为大额交易，客户通过第三方支付时常遇到问题，只能通过离线转账来完成交易。该页面用于展示转账信息并方便客户上传转账凭据。完成上传后客服会收到邮件通知，可迅速响应。

状态：已上线

0 元抽奖活动 (Sketch, MindNode)

为了提升 app 活跃度而策划的打卡抽奖活动，包含 PC web，mobile web，app 页面，其中 app 页面计划和 mweb 使用同一页面 (Hybrid 技术)，所以实际上的研发任务是两个 web 页面。活动管理后台已在规划中。

状态：离职时尚未开发

APP 版本管理后台 (Axure)

该后台用于管理 iOS/android 线上版本号和版本描述、上传最新版安卓 apk 包，并提供了是否允许跳过更新的开关。原型里同时规定了 app 内弹窗的样式。

状态：离职时尚未开发
